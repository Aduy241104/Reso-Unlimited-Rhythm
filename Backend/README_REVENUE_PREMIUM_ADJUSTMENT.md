# Revenue & Premium Model Adjustment Guide

## Goal

Adjust the current models to support:

- User purchasing Premium plans
- Admin viewing monthly Premium revenue
- Artist viewing monthly revenue
- Artist viewing revenue by track in each month
- Artist requesting withdrawal
- Admin managing artist withdrawal requests

Premium-related models already exist, so do not create new models for Premium unless necessary.

Existing Premium models:

- `Plan`
- `Subscription`
- `Transaction`
- `User.subscription`

The main adjustment should focus on the Revenue module.

---

## Current Existing Models To Reuse

### Premium

Use existing models:

```text
Plan
Subscription
Transaction
User.subscription
```

`Transaction` is the source of Premium revenue.

Monthly Premium revenue should be calculated from:

```js
Transaction.status === "success"
Transaction.paidAt within selected month
```

Do not create a new `PaymentHistory` model because `Transaction` already stores payment history.

---

## Models To Adjust / Add

Only make these changes:

```text
1. Update TrackMonthlyStat
2. Add ArtistRevenueSummary
3. Add WithdrawalRequest
4. Update PlatformMonthlyStat
```

Do not create `TrackRevenueSummary` because track revenue should be stored inside `TrackMonthlyStat`.

---

# 1. Update TrackMonthlyStat

## Purpose

`TrackMonthlyStat` currently stores monthly statistics of each track.

Add a `revenue` object to store monthly revenue generated by each track.

## Required Fields

```js
revenue: {
    eligibleStreams: {
        type: Number,
        default: 0,
        min: 0
    },

    grossRevenueAmount: {
        type: Number,
        default: 0,
        min: 0
    },

    artistRevenueAmount: {
        type: Number,
        default: 0,
        min: 0
    },

    platformRevenueAmount: {
        type: Number,
        default: 0,
        min: 0
    },

    revenueSharePercent: {
        type: Number,
        default: 0,
        min: 0
    },

    calculatedAt: {
        type: Date,
        default: null
    }
}
```

## Meaning

```text
eligibleStreams        = valid streams used for revenue calculation
grossRevenueAmount     = total revenue generated by this track
artistRevenueAmount    = amount artist receives from this track
platformRevenueAmount  = amount platform keeps from this track
revenueSharePercent    = track's percentage in the monthly revenue pool
calculatedAt           = time when revenue was calculated
```

## Indexes

Keep existing unique index:

```js
TrackMonthlyStatSchema.index(
    { trackId: 1, year: 1, month: 1 },
    { unique: true }
);
```

Add this index:

```js
TrackMonthlyStatSchema.index(
    { year: 1, month: 1, "revenue.artistRevenueAmount": -1 }
);
```

# 2. Add ArtistRevenueSummary Model

## Purpose

Stores monthly total revenue of each artist.

This model is used to display:

- Artist monthly revenue
- Artist revenue history
- Artist available balance
- Admin artist revenue list
- Admin revenue distribution history

## Model

```js
import mongoose from "mongoose";

const { Schema, model } = mongoose;

const ArtistRevenueSummarySchema = new Schema(
    {
        artistId: {
            type: Schema.Types.ObjectId,
            ref: "Artist",
            required: true,
            index: true
        },

        year: {
            type: Number,
            required: true,
            min: 2000
        },

        month: {
            type: Number,
            required: true,
            min: 1,
            max: 12
        },

        totalEligibleStreams: {
            type: Number,
            default: 0,
            min: 0
        },

        grossRevenueAmount: {
            type: Number,
            default: 0,
            min: 0
        },

        artistRevenueAmount: {
            type: Number,
            default: 0,
            min: 0
        },

        platformRevenueAmount: {
            type: Number,
            default: 0,
            min: 0
        },

        withdrawnAmount: {
            type: Number,
            default: 0,
            min: 0
        },

        availableAmount: {
            type: Number,
            default: 0,
            min: 0
        },

        status: {
            type: String,
            enum: ["pending", "calculated", "paid"],
            default: "pending",
            index: true
        },

        calculatedAt: {
            type: Date,
            default: null
        }
    },
    { timestamps: true }
);

ArtistRevenueSummarySchema.index(
    { artistId: 1, year: 1, month: 1 },
    { unique: true }
);

ArtistRevenueSummarySchema.index(
    { year: 1, month: 1 }
);

export default model("ArtistRevenueSummary", ArtistRevenueSummarySchema);
```

## Meaning

```text
totalEligibleStreams   = total valid streams of all tracks by this artist in the month
grossRevenueAmount     = total revenue generated by this artist before platform split
artistRevenueAmount    = amount artist receives
platformRevenueAmount  = amount platform keeps
withdrawnAmount        = amount already withdrawn by artist
availableAmount        = remaining amount artist can withdraw
status                 = revenue calculation/payment status
calculatedAt           = time when monthly revenue was calculated
```

## How To Calculate Artist Balance

Artist remaining balance should be calculated from:

```js
SUM(ArtistRevenueSummary.availableAmount)
```

Example aggregation:

```js
db.artistRevenueSummaries.aggregate([
    {
        $match: {
            artistId: artistId
        }
    },
    {
        $group: {
            _id: "$artistId",
            totalRevenueAmount: { $sum: "$artistRevenueAmount" },
            totalWithdrawnAmount: { $sum: "$withdrawnAmount" },
            totalAvailableAmount: { $sum: "$availableAmount" }
        }
    }
]);
```

# 3. Add WithdrawalRequest Model

## Purpose

Stores artist withdrawal requests and admin processing status.

This model is used for:

- Artist creating withdrawal request
- Artist viewing withdrawal history
- Admin viewing withdrawal requests
- Admin approving withdrawal request
- Admin rejecting withdrawal request
- Admin marking withdrawal as paid

## Model

```js
import mongoose from "mongoose";

const { Schema, model } = mongoose;

const WithdrawalRequestSchema = new Schema(
    {
        artistId: {
            type: Schema.Types.ObjectId,
            ref: "Artist",
            required: true,
            index: true
        },

        amount: {
            type: Number,
            required: true,
            min: 0
        },

        method: {
            type: String,
            enum: ["bank", "momo"],
            required: true,
            index: true
        },

        accountInfo: {
            bankName: {
                type: String,
                trim: true,
                default: ""
            },

            accountNumber: {
                type: String,
                trim: true,
                default: ""
            },

            accountHolderName: {
                type: String,
                trim: true,
                default: ""
            }
        },

        status: {
            type: String,
            enum: ["pending", "approved", "rejected", "paid"],
            default: "pending",
            index: true
        },

        requestedAt: {
            type: Date,
            default: Date.now,
            index: true
        },

        processedBy: {
            type: Schema.Types.ObjectId,
            ref: "User",
            default: null
        },

        processedAt: {
            type: Date,
            default: null
        },

        adminNote: {
            type: String,
            default: ""
        },

        rejectReason: {
            type: String,
            default: ""
        }
    },
    { timestamps: true }
);

WithdrawalRequestSchema.index(
    { artistId: 1, status: 1 }
);

WithdrawalRequestSchema.index(
    { status: 1, requestedAt: -1 }
);

export default model("WithdrawalRequest", WithdrawalRequestSchema);
```

## Important Rule

When a withdrawal is marked as `paid`, update artist revenue balance:

```js
ArtistRevenueSummary.withdrawnAmount += amount
ArtistRevenueSummary.availableAmount -= amount
```

If the withdrawal amount is greater than the artist's available balance, reject the request.

# 4. Update PlatformMonthlyStat

## Purpose

`PlatformMonthlyStat` already stores monthly platform statistics.

Add `revenueStats` to allow admin to view monthly revenue snapshot.

## Required Fields

```js
revenueStats: {
    totalPremiumRevenue: {
        type: Number,
        default: 0,
        min: 0
    },

    totalArtistPool: {
        type: Number,
        default: 0,
        min: 0
    },

    totalPlatformRevenue: {
        type: Number,
        default: 0,
        min: 0
    },

    totalSuccessfulTransactions: {
        type: Number,
        default: 0,
        min: 0
    },

    totalRefundedAmount: {
        type: Number,
        default: 0,
        min: 0
    }
}
```

## Meaning

```text
totalPremiumRevenue          = total successful Premium transaction amount in month
totalArtistPool              = total amount used to pay artists
totalPlatformRevenue         = amount kept by platform
totalSuccessfulTransactions  = count of successful Premium transactions
totalRefundedAmount          = refunded amount in month
```

## Revenue Source

Monthly revenue must be calculated from `Transaction`.

Condition:

```js
status: "success"
paidAt: {
    $gte: startOfMonth,
    $lte: endOfMonth
}
```

Example:

```js
const monthlyRevenue = await Transaction.aggregate([
    {
        $match: {
            status: "success",
            paidAt: {
                $gte: startOfMonth,
                $lte: endOfMonth
            }
        }
    },
    {
        $group: {
            _id: null,
            totalPremiumRevenue: { $sum: "$totalAmount" },
            totalBeforeTax: { $sum: "$amount" },
            totalTax: { $sum: "$tax" },
            totalSuccessfulTransactions: { $sum: 1 }
        }
    }
]);
```

# Revenue Calculation Flow

## Step 1: Calculate Monthly Premium Revenue

Source:

```text
Transaction
```

Filter:

```text
status = success
paidAt is within selected month
```

Result:

```text
totalPremiumRevenue
```

---

## Step 2: Calculate Revenue Pool

Example fixed rule:

```text
artistPool = totalPremiumRevenue * 70%
platformRevenue = totalPremiumRevenue * 30%
```

For now, do not create `RevenueSetting`.

Use constants in service layer if needed.

---

## Step 3: Calculate Track Revenue

Source:

```text
TrackMonthlyStat
```

Use:

```text
eligibleStreams
```

Formula:

```text
trackRevenue = artistPool * (trackEligibleStreams / totalEligibleStreamsOfPlatform)
```

Update:

```text
TrackMonthlyStat.revenue
```

---

## Step 4: Calculate Artist Revenue

Artist revenue is the sum of all track revenue from that artist in the month.

Formula:

```text
artistRevenue = SUM(TrackMonthlyStat.revenue.artistRevenueAmount)
```

Update:

```text
ArtistRevenueSummary
```

---

## Step 5: Update Platform Monthly Revenue Snapshot

Update:

```text
PlatformMonthlyStat.revenueStats
```

# Final Model Decision

Do not add too many models.

Use this final structure:

```text
Premium
├── Plan                         existing
├── Subscription                 existing
├── Transaction                  existing
└── User.subscription            existing

Revenue
├── TrackMonthlyStat             update: add revenue
├── ArtistRevenueSummary         new
├── WithdrawalRequest            new
└── PlatformMonthlyStat          update: add revenueStats
```

Do not create:

```text
TrackRevenueSummary
RevenueSetting
PaymentHistory
ArtistBalance
```

Reason:

```text
TrackRevenueSummary  -> merged into TrackMonthlyStat
RevenueSetting       -> not needed now
PaymentHistory       -> Transaction already handles this
ArtistBalance        -> calculate from ArtistRevenueSummary.availableAmount
```
